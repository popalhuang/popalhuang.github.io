Solace Install for VitualBox

  1. Get a Software Event Broker
    1. Solace PubSub+ Standard or Solace PubSub+ Enterprise Evaluation Edition
    2. Solace PubSub+ Enterprise
  2. Import the Software Event Broker
    1. In Oracle VM VirtualBox Manager, select File > Import Appliance.
    2. select the OVA file downloaded in the previous step
    3. default appliance settings are sufficient to begin the import
    4. Oracle VM VirtualBox Manager will import the event broker OVA.
    5. VM VirtualBox Manager,Settings
    6. start the VM
    7. Login to the event broker,must “sysadmin”
  3. Configure the Host Name
    1. Shut down Solace PubSub+
    2. Configure the hostname
    3. Restart the Solace PubSub+
    4. Verify that the hostname has been set to the new one
  4. Access the Solace CLI
    1. To enter the Solace CLI from the console in the Linux host environment
    2. create an admin user named
    3. To determine the IP address assigned to the event broker
    4. To remotely access the Solace CLI for the event broker, you can now ssh to port 2222
  5. Review Configuration Defaults

Hadoop 3.0.0 Overview

  1. Minimum required Java version increased from Java 7 to Java 8
  2. Support for erasure coding in HDFS
  3. YARN Timeline Service v.2
  4. Shell script rewrite
  5. Shaded client jars
  6. Support for Opportunistic Containers and Distributed Scheduling.
  7. MapReduce task-level native optimization
  8. Support for more than 2 NameNodes
  9. Default ports of multiple services have been changed.
  10. Support for Microsoft Azure Data Lake and Aliyun Object Storage System filesystem connectors
  11. Intra-datanode balancer
  12. Reworked daemon and task heap management
  13. S3Guard: Consistency and Metadata Caching for the S3A filesystem client
  14. HDFS Router-Based Federation
  15. API-based configuration of Capacity Scheduler queue configuration
  16. YARN Resource Types
